	\chapter{Antecedentes}}

Buscando información no se encontró un estudio previo en español acerca de la implementación CAN en un chip a nivel doctorado. Aunque existen algunas tesis de maestría acerca del uso de este protocolo. Por ejemplo, encontré una tesis de maestría de Edilberto Lopez Perez.titulada ESPECIFICACION FORMAL DEL PROTOCOLO DE COMUNICACIONES CAN FD (Flexible Data Rate) MEDIANTE SDL (Specification and Description Language). Donde se especifica el comportamiento este protocolo. Para ello se usa un lenguaje de diagramas de alto nivel tipo UML que permite describir la jerarquía del sistema. Pero solo se realiza la simulación de este y no se implementa ni se hacen pruebas en una red CAN real.
 
\section{Estudio de Red CAN}
También encontré otra tesis de maestría de Ramon Ariel Vela Xool. DESARROLLO SISTEMA PARA MONITOREO E INTERACCION CON SEÑALES DE BUS CAN. En este trabajo se muestra como implementar un controlador CAN con un Arduino y una librería para la comunicación con este protocolo. Así mismo se monitorean los datos desde una computadora en una red CAN de dos nodos. Esta podría consultarse más adelante para ver como establecer una red.

\section{Estudio de CAN ASIC}
También encontré una tesis de maestría de SREERAM KRISHNAMOORTHY, llamada DESIGN OF AN ASIC CHIP FOR A CONTROLLER AREA NETWORK (CAN) PROTOCOL CONTROLLER. En esta tesis se implementa el protocolo CAN a nivel RTL (register transfer level) en un chip. La diferencia es que no usa herramientas open source sino otras llamadas Magma tools para la realización física del diseño y Cadence Verplex para la verificación lógica. Este trabajo sin embargo puede resultar útil para basarme en cómo se logró la implementación de este protocolo y estudiar el diagrama de bloques del sistema implementado en un chip.

\section{Documento de C\_CAN}
Otro documento importante en el que puedo apoyar mi trabajo es el manual del usuario C\_CAN de la compañía Bosch. De acuerdo a este, C\_CAN es un módulo escrito en VHDL que está listo para síntesis (es decir se puede convertir a un chip). Aunque no muestra ese código, trae la descripción del sistema por medio de diagramas de bloques. También explica como configurar y que función tienen cada uno de los registros que tendrá este chip. Sin embargo, no menciona que herramientas se pueden usar o como hacer la síntesis.

\section{Estudio de Macro Memory Cell Generator}
Otro trabajo interesante es el de Uriel Jaramillo Toral llamado Macro Memory Cell Generator for SKY130 PDK. En esta publicación se muestra el uso de OpenLane para realizar configuraciones de memoria SRAM según sea las necesidades del diseñador. Lo interesante es que usa la misma plataforma que se pretende usar para este proyecto de modo que se puede aprovechar la experiencia en el uso de esta herramienta.

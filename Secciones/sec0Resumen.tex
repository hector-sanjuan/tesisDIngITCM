\fontsize{14}{14}\selectfont \textbf{Resumen}
\end{center}

El protocolo CAN sirve para enviar y recibir información de forma eficiente entre los diversos nodos o componentes de un sistema automotriz. Es ampliamente usado en la industria automotriz y existen diversas versiones como CAN FD y CAN XL que permiten comunicación Full duplex y un formato extendido respectivamente. La especificación original ofrece funcionalidad a bajo nivel en las capas del modelo OSI, que son las capas físicas y de data link. Este protocolo ofrece varias oportunidades para ser mejorado y extendido en cualquiera de las capas superiores de ese modelo. Por ejemplo, protocolos de seguridad, estampas de tiempo, activación por tiempo y para lograr un ancho de banda mas amplio. Todas estas nuevas características están implementadas en capas de superiores, es decir, de red, transporte, sesión y presentación. Se presenta la implementación de un nuevo modulo de propiedad intelectual que mejora el protocolo de comunicación CAN. Esto ayuda a poder transmitir datos junto con otro protocolo como MQTT o poder transmitir información de manera que sea posible configurar el modulo a través de alguna aplicación o interfaz remota. También se considera procesar información de diagnósticos o aumentar la seguridad del protocolo a través de encriptación. Esta mejora tiene el mismo desempeño que un modulo CAN normal sin modificar implicando que dicha mejora no supone una sobrecarga. Al mismo tiempo, se hace uso de la tecnología open source para crear un circuito integrado de propósito especifico (ASIC) que implemente dicha funcionalidad. Esto se logra a través del diseño de hardware en lenguaje Verilog y generando los archivos necesarios llamados GDSII. Este archivo se usa para poder manufacturar el circuito integrado. Esto demuestra la efectividad de esta herramienta como alternativa a las herramientas de propietarias de gran costo. De esta manera se ofrece un diseño físico al final del proyecto operando en una red CAN real.
